## General Onboarding

### Access & Tools
- [ ] Added to project communication channels (Slack, Teams, etc.)
- [ ] Access to project board (GitHub Projects, Jira, etc.)
- [ ] Access to repository/repositories
- [ ] Access to documentation wiki or shared drive
- [ ] Added to calendar invites for recurring meetings
- [ ] Access to monitoring/observability tools (if applicable)

### Project Context
- [ ] Read Project One-pager (see [Project Initiation Guide](./octoacme-project-initiation.md))

### Process Understanding
- [ ] Review [Project Planning](./octoacme-project-planning.md) process
- [ ] Review [Execution & Tracking](./octoacme-execution-and-tracking.md) practices
- [ ] Understand Definition of Done for this project
- [ ] Review PR workflow and branching conventions

- [ ] Review [Risk Management & Communication](./octoacme-risks-and-communication.md) practices
- [ ] Know escalation paths and incident response procedures

## Role-Specific Onboarding

### For Developers
- [ ] Set up local development environment
- [ ] Run tests successfully locally
- [ ] Review architecture docs and technical design documents
- [ ] Pair with a team member on first PR
- [ ] Understand code review expectations
- [ ] Know how to run security scanning and linting locally

### For Project Managers
- [ ] Access to risk register and project status dashboard
- [ ] Scheduled 1:1 with Product Manager and Tech Lead
- [ ] Understand reporting cadence and stakeholder expectations
- [ ] Review existing project timeline and milestones
- [ ] Access to previous meeting notes and decision logs
- [ ] Know how to coordinate releases (if applicable)

### For Product Managers
- [ ] Access to product analytics and user research data
- [ ] Review roadmap and backlog prioritization
- [ ] Scheduled 1:1s with Project Manager and key stakeholders
- [ ] Understand customer personas and use cases
- [ ] Access to customer feedback channels
- [ ] Know success metrics and how they're tracked

### For Scrum Masters
- [ ] Review team's current agile practices and ceremonies
- [ ] Understand team capacity and velocity
- [ ] Access to retrospective notes and action items
- [ ] Schedule 1:1s with team members
- [ ] Identify current blockers and impediments
- [ ] Review team working agreements

### For UX Designers
- [ ] Access to design tools and design system
- [ ] Review user research findings and personas
- [ ] Understand design review process
- [ ] Access to user testing platform (if applicable)
- [ ] Review accessibility requirements and standards
- [ ] Schedule sync with Product Manager and Tech Lead

### For QA/Test Engineers
- [ ] Access to test environments
- [ ] Review test plan and test cases
- [ ] Understand CI/CD pipeline and automated testing
- [ ] Access to bug tracking system
- [ ] Review acceptance criteria format and expectations
- [ ] Know release sign-off process

### For Release Managers
- [ ] Access to deployment pipelines and production systems
- [ ] Review deployment runbooks and rollback procedures
- [ ] Understand release cadence and windows
- [ ] Access to on-call schedule and incident response process
- [ ] Review monitoring and alerting setup
- [ ] Know stakeholder notification process for releases

### For Technical Writers
- [ ] Access to documentation platform and CMS
- [ ] Review documentation style guide and standards
- [ ] Understand documentation review process
- [ ] Access to product/feature roadmap
- [ ] Identify documentation gaps and priorities
- [ ] Schedule syncs with PM, PdM, and developers

### For Customer Success
- [ ] Access to customer communication platforms
- [ ] Review customer list and account health metrics
- [ ] Understand customer escalation process
- [ ] Access to product usage analytics
- [ ] Schedule 1:1s with Product Manager and Support Lead
- [ ] Review customer feedback mechanisms

## First Week Goals

### Days 1-2: Orientation
- [ ] Complete access and tools setup
- [ ] Read all project documentation
- [ ] Meet with Project Manager for project overview
- [ ] Meet key team members

### Days 3-5: Engagement
- [ ] Attend first sprint ceremonies (standup, planning, demo, etc.)
- [ ] Shadow a team member in your role
- [ ] Contribute to first work item or review
- [ ] Ask questions and clarify expectations

### End of Week 1
- [ ] Complete onboarding checklist
- [ ] Schedule check-in with manager or PM
- [ ] Identify any gaps in documentation or process
- [ ] Feel comfortable with team communication channels

---

## Ongoing Integration (Weeks 2-4)
- [ ] Take on increasing responsibility in role
- [ ] Provide feedback on onboarding experience
- [ ] Identify improvement opportunities
- [ ] Build relationships across the team
